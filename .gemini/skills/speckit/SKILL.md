# Speckit Skill

This skill guides the EditaisFinder project through the **Spec-Driven Development (SDD)** lifecycle.

## Workflow Commands

- **/speckit.constitution**: Reference the project's foundational rules. Read [references/constitution.md](references/constitution.md).
- **/speckit.specify**: Define a new feature or fix. Create a specification file in `.specify/specs/`.
- **/speckit.clarify**: Analyze the spec for ambiguities. Ask clarifying questions before planning.
- **/speckit.plan**: Design the technical solution. Document the tech stack and architectural choices.
- **/speckit.tasks**: Break the plan into atomic, testable tasks.
- **/speckit.implement**: Execute tasks using the Red-Green-Refactor loop.

## Core Reference
Always refer to the [Constitution](references/constitution.md) for non-negotiable project standards.
